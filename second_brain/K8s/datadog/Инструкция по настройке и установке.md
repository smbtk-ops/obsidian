# Datadog Agent для Kubernetes

## Требования

- Kubernetes 1.21+
- Helm 3.x
- API ключ Datadog

## Установка

### 1. Добавить Helm репозиторий

```bash
helm repo add datadog https://helm.datadoghq.com
helm repo update
```

### 2. Создать namespace

```bash
kubectl create namespace datadog
```

### 3. Создать secret с API ключом

```bash
kubectl create secret generic datadog-secret \
  --from-literal=api-key=<YOUR_API_KEY> \
  -n datadog
```

### 4. Установить Datadog Agent

```bash
helm install datadog-agent datadog/datadog \
  -n datadog \
  -f values.yaml \
  --set datadog.apiKeyExistingSecret=datadog-secret
```

Или с API ключом напрямую (не рекомендуется для production):

```bash
helm install datadog-agent datadog/datadog \
  -n datadog \
  -f values.yaml \
  --set datadog.apiKey=<YOUR_API_KEY>
```

## Обновление

```bash
helm repo update
helm upgrade datadog-agent datadog/datadog \
  -n datadog \
  -f values.yaml \
  --set datadog.apiKeyExistingSecret=datadog-secret
```

## Проверка установки

### Статус подов

```bash
kubectl -n datadog get pods
```

Ожидаемый результат:
- `datadog-agent-xxxxx` - DaemonSet, по одному поду на каждую ноду
- `datadog-agent-cluster-agent-xxxxx` - 2 реплики для HA

### Статус агента

```bash
kubectl -n datadog exec -it daemonset/datadog-agent -c agent -- agent status
```

### Логи агента

```bash
kubectl -n datadog logs daemonset/datadog-agent -c agent --tail=50
```

## Конфигурация

### Основные параметры в values.yaml

| Параметр | Описание | Значение по умолчанию |
|----------|----------|----------------------|
| `datadog.site` | Регион Datadog | `datadoghq.eu` |
| `datadog.tags` | Глобальные теги | `env:production` |
| `datadog.logs.enabled` | Сбор логов | `true` |
| `datadog.logs.containerCollectAll` | Логи со всех контейнеров | `true` |
| `datadog.apm.portEnabled` | APM через порт 8126 | `true` |
| `datadog.apm.socketEnabled` | APM через Unix socket | `true` |
| `datadog.processAgent.enabled` | Мониторинг процессов | `true` |
| `clusterAgent.replicas` | Количество реплик Cluster Agent | `2` |

### Мониторинг control-plane нод

Для мониторинга master нод добавить tolerations:

```yaml
agents:
  tolerations:
    - key: node-role.kubernetes.io/control-plane
      operator: Exists
      effect: NoSchedule
```

### Фильтрация логов по namespace

Собирать логи только из определенных namespace:

```yaml
datadog:
  logs:
    enabled: true
    containerCollectAll: false
  containerExcludeLogs: "kube_namespace:kube-system kube_namespace:metallb-system"
```

Или включить только определенные:

```yaml
datadog:
  logs:
    enabled: true
    containerCollectAll: false
  containerIncludeLogs: "kube_namespace:slotegrator kube_namespace:maxline"
```

### Network Performance Monitoring

```yaml
datadog:
  networkMonitoring:
    enabled: true
```

### Live Processes

```yaml
datadog:
  processAgent:
    enabled: true
    processCollection: true
```

## Интеграция с приложениями

### APM для приложений

Приложения могут отправлять трейсы на:
- **TCP**: `datadog-agent.datadog.svc.cluster.local:8126`
- **Unix socket**: `/var/run/datadog/apm.socket`

Пример для Python:

```python
from ddtrace import tracer

tracer.configure(
    hostname="datadog-agent.datadog.svc.cluster.local",
    port=8126,
)
```

### DogStatsD метрики

Приложения могут отправлять метрики на:
- **UDP**: `datadog-agent.datadog.svc.cluster.local:8125`
- **Unix socket**: `/var/run/datadog/dsd.socket`

### Настройка сбора логов через аннотации

Аннотации позволяют точно настроить сбор логов для каждого приложения.

#### Формат аннотации

```yaml
annotations:
  ad.datadoghq.com/<container-name>.logs: |
    [{
      "source": "<тип логов>",
      "service": "<имя сервиса>",
      "tags": ["tag1:value1"]
    }]
```

| Параметр | Описание |
|----------|----------|
| `<container-name>` | Имя контейнера в поде |
| `source` | Тип логов для парсера (go, java, python, nginx, nodejs и др.) |
| `service` | Имя сервиса в интерфейсе Datadog |
| `tags` | Дополнительные теги (опционально) |

#### Примеры для приложений

Go-приложение (log-microservice):

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: log-microservice
  namespace: maxline
spec:
  template:
    metadata:
      annotations:
        ad.datadoghq.com/log-microservice.logs: |
          [{"source": "go", "service": "maxline-logger"}]
```

Java-приложение (slotegrator):

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: slotegrator
  namespace: slotegrator
spec:
  template:
    metadata:
      annotations:
        ad.datadoghq.com/slotegrator.logs: |
          [{"source": "java", "service": "slotegrator"}]
```

#### Доступные source типы

| source | Язык/Приложение |
|--------|-----------------|
| `go` | Go |
| `java` | Java |
| `python` | Python |
| `nodejs` | Node.js |
| `nginx` | Nginx |
| `postgresql` | PostgreSQL |
| `redis` | Redis |
| `docker` | Общий формат Docker |

#### Исключение контейнера из сбора логов

```yaml
annotations:
  ad.datadoghq.com/<container-name>.logs: "false"
```

### APM auto-instrumentation

Включить автоматическую инструментацию для трейсинга:

```yaml
labels:
  admission.datadoghq.com/enabled: "true"
annotations:
  # Для Java
  admission.datadoghq.com/java-lib.version: "latest"
  # Для Python
  # admission.datadoghq.com/python-lib.version: "latest"
  # Для Node.js
  # admission.datadoghq.com/js-lib.version: "latest"
```

## Удаление

```bash
helm uninstall datadog-agent -n datadog
kubectl delete namespace datadog
```
