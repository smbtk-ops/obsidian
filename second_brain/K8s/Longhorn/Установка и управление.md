## 1. Что такое Longhorn

Longhorn — distributed block storage system для Kubernetes.

---

## 2. Prerequisites

### 2.1. Установить зависимости на нодах

```bash
cd k8s-ansible
source ../kubespray/venv/bin/activate
ansible-playbook 00_prepare_prod_host.yml --tags=longhorn -u ansible
```

### 2.2. Дополнительная настройка ядра

Для устранения предупреждений в Longhorn UI (KernelModulesLoaded, Multipathd):

```bash
# Загрузить модуль dm_crypt (для шифрования томов)
ansible workers -m shell -a "modprobe dm_crypt && echo dm_crypt >> /etc/modules-load.d/dm_crypt.conf" -u ansible --become

# Отключить multipathd (если не используется SAN storage)
ansible workers -m shell -a "systemctl disable --now multipathd.socket multipathd.service" -u ansible --become
```

### 2.3. Подготовить storage на воркерах

На воркерах создан отдельный LVM том для Longhorn:

| Параметр           | Значение  |
| ------------------ | --------- |
| Volume Group       | ubuntu-vg |
| Logical Volume     | data-lv   |
| Размер             | 200G      |
| Файловая система   | ext4      |
| Точка монтирования | /mnt/data |

### 2.4. Создание storage (если не выполнено)

```bash
# Создать LV
ansible workers -m shell -a "lvcreate -L 200G -n data-lv ubuntu-vg" -u ansible --become

# Создать файловую систему
ansible workers -m shell -a "mkfs.ext4 /dev/ubuntu-vg/data-lv" -u ansible --become

# Создать директорию и смонтировать
ansible workers -m shell -a "mkdir -p /mnt/data && mount /dev/ubuntu-vg/data-lv /mnt/data" -u ansible --become

# Добавить в fstab
ansible workers -m shell -a "echo '/dev/ubuntu-vg/data-lv /mnt/data ext4 defaults 0 2' >> /etc/fstab" -u ansible --become

# Проверить
ansible workers -m shell -a "df -h /mnt/data" -u ansible --become
```

### 2.5. Расширение storage

Если нужно увеличить размер тома (например, добавить ещё 100G):

```bash
# Расширить LV
ansible workers -m shell -a "lvextend -L +100G /dev/ubuntu-vg/data-lv" -u ansible --become

# Расширить файловую систему
ansible workers -m shell -a "resize2fs /dev/ubuntu-vg/data-lv" -u ansible --become

# Проверить новый размер
ansible workers -m shell -a "df -h /mnt/data" -u ansible --become
```

Доступное место в VG можно проверить:

```bash
ansible workers -m shell -a "vgs" -u ansible --become
```

## 3. Установка Longhorn

### 3.1. Добавить Helm репозиторий

```bash
helm repo add longhorn https://charts.longhorn.io
helm repo update
```

### 3.2. Отредактировать values.yaml

Основные параметры:

```yaml
defaultSettings:
  # Путь для хранения данных
  defaultDataPath: /mnt/data
  # Количество реплик (2 = переживёт отказ 1 ноды)
  defaultReplicaCount: 2
```

### 3.3. Установить Longhorn

```bash
kubectl create namespace longhorn-system
helm install longhorn longhorn/longhorn \
  --namespace longhorn-system \
  --values values.yaml
```

### 3.4. Проверить статус

```bash
kubectl -n longhorn-system get pods
kubectl -n longhorn-system get svc
```

## 4. Доступ к UI

### 4.1. Port-forward (рекомендуется)

Безопасный способ доступа — трафик идёт только через localhost.

```bash
# Запустить port-forward
kubectl -n longhorn-system port-forward svc/longhorn-frontend 8080:80
```

Открыть http://localhost:8080

Запуск в фоне:

```bash
# Запустить в фоне
kubectl -n longhorn-system port-forward svc/longhorn-frontend 8080:80 &

# Остановить
pkill -f "port-forward.*longhorn-frontend"
```

**Важно:** Longhorn UI не имеет встроенной аутентификации. Port-forward безопасен, так как доступен только с локальной машины.

### 4.2. Ingress (опционально)

Для постоянного доступа можно настроить Ingress. **Обязательно** добавить аутентификацию (basic auth, OAuth).

```yaml
ingress:
  enabled: true
  host: longhorn.example.com
  annotations:
    nginx.ingress.kubernetes.io/auth-type: basic
    nginx.ingress.kubernetes.io/auth-secret: longhorn-basic-auth
```

## 5. StorageClass

При установке создаются два StorageClass:

| StorageClass           | Назначение                                    |
| ---------------------- | --------------------------------------------- |
| `longhorn` (default)   | Динамическое создание томов через PVC         |
| `longhorn-static`      | Подключение существующих томов вручную        |

### 5.1. Использование longhorn (стандартный способ)

```yaml
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: my-pvc
spec:
  accessModes:
    - ReadWriteOnce
  storageClassName: longhorn
  resources:
    requests:
      storage: 10Gi
```

### 5.2. Использование longhorn-static

Для подключения уже существующего тома (восстановленного из бэкапа, созданного через UI, или импортированного).

**Шаг 1: Получить Volume Handle**

Найти ID тома в UI Longhorn или через CLI:

```bash
kubectl -n longhorn-system get volumes.longhorn.io
```

Пример вывода:

```
NAME                                       STATE     ROBUSTNESS   SCHEDULED   SIZE          NODE     AGE
pvc-a1b2c3d4-e5f6-7890-abcd-ef1234567890   attached  healthy      True        10737418240   worker1  1d
```

Volume Handle = `pvc-a1b2c3d4-e5f6-7890-abcd-ef1234567890`

**Шаг 2: Создать PersistentVolume**

```yaml
apiVersion: v1
kind: PersistentVolume
metadata:
  name: my-restored-pv
spec:
  capacity:
    storage: 10Gi
  volumeMode: Filesystem
  accessModes:
    - ReadWriteOnce
  persistentVolumeReclaimPolicy: Retain
  storageClassName: longhorn-static
  csi:
    driver: driver.longhorn.io
    fsType: ext4
    volumeHandle: pvc-a1b2c3d4-e5f6-7890-abcd-ef1234567890
```

**Шаг 3: Создать PersistentVolumeClaim**

```yaml
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: my-restored-pvc
spec:
  accessModes:
    - ReadWriteOnce
  storageClassName: longhorn-static
  resources:
    requests:
      storage: 10Gi
  volumeName: my-restored-pv
```

### 5.3. Типичные сценарии использования longhorn-static

1. **Восстановление из бэкапа** — создать том из бэкапа в UI, затем подключить через static
2. **Миграция данных** — перенос тома между namespace или кластерами
3. **Клонирование** — создать snapshot, из него том, подключить к другому поду
4. **Отладка** — подключить существующий том к временному поду для диагностики

## 6. Расширение томов

Longhorn поддерживает online expansion — расширение без остановки пода.

### 6.1. Расширить PVC

```bash
# Через patch
kubectl patch pvc my-pvc -p '{"spec":{"resources":{"requests":{"storage":"100Gi"}}}}'

# Или через edit
kubectl edit pvc my-pvc
# изменить spec.resources.requests.storage: 50Gi → 100Gi
```

### 6.2. Что произойдёт

1. Longhorn расширит volume автоматически
2. Файловая система расширится автоматически
3. Pod продолжает работать, даунтайма нет

### 6.3. Ограничения

- Уменьшить размер нельзя (только увеличить)
- Должно быть свободное место в кластере Longhorn
